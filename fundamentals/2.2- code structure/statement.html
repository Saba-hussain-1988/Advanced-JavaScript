CODE STRUCTURE

The first thing, We'll study is the building blocks of code.

STATEMENTS

Statements are syntax constructs and commands that perform actions.
Statements can be separated with a semicolon.
Usually, statements are written in separate lines to make the code more readable.

SEMICOLONS

Semicolon may be omitted in most cases when a line break exist.
Java Script interprets the line break as an "implicit" semicolon.
This is called "Automatic Semicolon Insertion".

In most cases, A new line implies a semicolon. But "in most cases" does not means always.

There are cases when a new line does not means a semicolon.
alert(2 +
3
+ 5)
It is intuitively obvious that it is an incomplete expression, So a semicolon there would be incorrect.
That works as intended.

But there are situations where Java Script "fails" to assume a semicolon where is really needed.

Errors which occur in such cases are quite hard to find and fix.
alert("Hello")
[1, 2].foreach(alert);
Java Script does not assume semicolon before square brackets[...], So, the code is treated as a single statement.(there's an error)

We recommend putting semicolons in between statements even if they are separated by a new lines

COMMENTS

Comments describe what the code does and why.
Comments can be put into any place of script, engine simply ignores them.

One-line comment start with two forward slash characters

Multiline comments start with a forward slash and an asterisk /* and end with an asterisk and a forward slash */.
